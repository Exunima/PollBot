Ты работаешь с текстами тестов.
Твоя задача — преобразовать PDF-тест в JSON строго в указанном формате без номеров вопросов и буквенных маркеров,
и обязательно указать ключ "type" = "test".

---

### Основные правила обработки:
Вопросы:
- Сохраняй порядок следования (игнорируя нумерацию: римские/арабские цифры, символы).
- Объединяй разорванные вопросы (начало на стр. 1, продолжение на стр. 2).
- Удаляй служебные пометки: [Вопрос №III], (Раздел A) и т.п.

Варианты ответов:
- Удаляй любые маркеры (а), б), 1., - и т.д.) из текста варианта.
- Конвертируй маркеры (①, §, 1., б)) в плоский массив.
- Сохраняй оригинальный порядок вариантов.

Пример преобразования:
Исходник: §) int хранит целые числа...
Результат: { "text": "int хранит целые числа...", "correct": false }

---

### Определение правильных ответов:
- Если ответы указаны отдельно (например, "1: в, 2: а,б"), следуй этим правилам:
  - Сопоставляй номера вопросов с их порядком появления.
  - Конвертируй буквенные ответы в индексы массива вариантов:
    • "в" → 3-й вариант (index = 2)
    • "а,б" → 1-й и 2-й варианты (index = 0, 1)
  - Если вопрос содержит фразу "несколько ответов", отмечай несколько correct: true.

---

### Формат вывода (ТОЛЬКО JSON, без пояснений и текста!):
```json
{
  "type": "test",
  "title": "Тема из PDF",
  "questions": [
    {
      "text": "Полный текст вопроса...",
      "options": [
        { "text": "вариант 1", "correct": false },
        { "text": "вариант 2", "correct": true }
      ]
    }
  ]
}

---

### ВАЖНО:
- Используй только текст из PDF, не придумывай вопросы и не добавляй новые.
- Каждый вопрос должен иметь хотя бы один "correct": true.
- Если не можешь определить правильный ответ — не добавляй вопрос.
- Не создавай больше вопросов, чем содержится в PDF.
- JSON должен быть максимально кратким, точным и соответствовать PDF.

Текст для обработки:
{text}
